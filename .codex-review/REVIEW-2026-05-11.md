# Codex 项目级 Review — 2026-05-11

## TL;DR
项目的底层方向是对的：8 实体 schema、Capability + Principal、Claim/Evidence、Provenance、Plugin/Skill/MCP、CJK typography 都不是空话，很多核心件已经落在代码里。最大风险不是“没做”，而是 **STATUS/ADR promote 的语气比实际 dogfood 证据更满**：ADR-0008、0012、0013、0014 都还有真实端到端缺口。当前最该警惕的是 long-horizon agent runtime：worker 仍有 stub 路径，quota/cancel 没 enforce，multi-agent dogfood spec 多数是 shape-only。

## 1. 第一性原理落地度
**Local-first：部分兑现。**  
编辑器确实有 IndexedDB 本地持久化，`packages/editor-core/src/sync/setup.ts:53-60` 用 `IndexeddbPersistence(room, handle.yDoc)`，这支撑离线 reload 保留编辑状态。但 repo 里没有 Service Worker / 明确离线状态闭环；sync-gateway 仍是服务端 WS hot path。导出路径存在，`apps/web/src/app/api/export/[docId]/[format]/route.ts:160-250` 覆盖 HTML/JATS/Markdown/Typst/PDF/DOCX，但当前实际导出内容优先走 `?content=<base64-pm-json>` 或空 doc fallback，`route.ts:113-124` 说明 PG snapshot 到 PM tree 的真实恢复仍未闭环。

**Markup-as-source / WYSIWYM：核心一致，但 source 还偏 PM JSON。**  
commit serializer 是实的：`packages/editor-core/src/commit.ts:56-63` 同时产出 PM steps、Yjs update、base state vector。渲染 emitter 也接 PM JSON：HTML 在 `packages/render-myst/src/html.ts:37-68`，Typst 在 `packages/render-typst/src/source-from-pm.ts:57-76`。但“Markup-as-source”对外更多是 PM JSON as canonical wire，而不是 MyST/Typst 文件作为真正 source of truth。

**AI 是协作者不是侧边栏：W6 后明显进步，但还没完整。**  
`AgentTrigger` 已经从 selection 抽 `passage/blockId/blockKind`，`packages/editor-core/src/extensions/agent-trigger.ts:108-157` 是实打实替代手动粘贴；`InlineAgentMenu` 监听编辑器 DOM 事件并 POST invoke，`apps/web/src/app/(app)/editor/[docId]/components/InlineAgentMenu.tsx:144-160`、`:204-246`。但 researcher/reviewer chip 仍标 `routeSupported: false`，`apps/web/src/lib/inline-agent-menu.ts:84-95`；`/api/agent/invoke` 只支持 `citation | inline-editor`，`apps/web/src/app/api/agent/invoke/route.ts:76-81`。所以“协作动作”在两个高频动作上成立，在 multi-agent 叙事上还没成立。

**CJK / 拉丁双语：相对扎实。**  
CJK punctuation boundary 已修，`packages/typography/src/cjk-spacing.ts:47-57` 把 CJK 标点与 Latin 的边界纳入 spacing。Typst emitter 设置 `lang/region/font`，`packages/render-typst/src/source-from-pm.ts:57-66`；HTML emitter也消费 font tokens，`packages/render-myst/src/html.ts:44-63`。短板在 UI 文案：不少地方仍是 slash 拼接，不是成熟双语信息架构。

**Provenance：数据层强，是项目最大胜点。**  
`persistProposalBatch` 是唯一集中写入入口之一，三表事务 + batch，`packages/ai-runtime/src/provenance-writer.ts:89-181`；accept/reject 追加 approvalChain，`:247-264`、`:314-331`。但 direct human edit / non-agent contribution 的 provenance 覆盖需要继续 audit，不能只看 agent proposal 路径。

**延迟预算：基本没有被真实验证。**  
有目标和 observability slow 标记，`apps/web/src/app/api/export/[docId]/[format]/route.ts:143-156`，但没有 keystroke <100ms、公式 <50ms、PDF <5s 的自动 bench。PDF 甚至允许 503 fallback，`route.ts:208-243`。这条目前是设计目标，不是工程事实。

## 3. 反模式 / 红线触碰
最严重的是 ADR-0008 红线：自主/long-horizon agent 没有 quota + cancel。`AgentJobStatus` 只有 `cancelled`，没有 `cancelling`，`apps/agent-worker/src/job-types.ts:84-90`；全仓库 grep 只看到 quota parse，没有 enforce。trade-off：现在先做 mock/shape 能推进 UI，但一旦开放 researcher/reviewer，成本失控和不可中断会变成 P0 运营事故。

第二是“Accepted with caveat”用得过宽。ADR-0013 的 resolver 存在，`packages/ai-runtime/src/providers/resolver.ts:83-151`，但主 invoke route 没接；ADR-0014 有 cross-ref unit，但不是 50-client real sync。建议把“contract pass / mock pass / real dogfood pass”在 STATUS 分成三列，不要混用 PASS。

第三是 core 边界总体守住了，但 web route 仍手动找 citation/inline-editor plugin，`apps/web/src/app/api/agent/invoke/route.ts:127-146`。这不是灾难，但 Phase 5 用户安装 plugin 后，route 级 hard wiring 会拖累扩展系统。

## 4. Phase 4 W6-W10 交付 verify
W6：inline menu 和 DOI direct 是实的。`agent-trigger.ts:108-157`、`InlineAgentMenu.tsx:204-246` 兑现了“selection → action”。但 researcher/reviewer chip 不通，不能把 W6 叙事扩展成完整 multi-agent。

W7：doc-store、provider contract、bulk ACL、batch provenance 都有代码。短板是 doc-store 仍明确保留 `yDoc` escape hatch，`packages/doc-store/src/types.ts:12-16`；这是必要 trade-off，但不要再说“1-2 周可切 CRDT”。

W8：macOS/Windows 走 UI 显式拦截，诚实度比 placeholder 好；但 README 仍写“三平台沙箱描述符”，`README.md:40-43`，容易被理解为三平台可用。

W9：5 dogfood gate CI 多数是 “contract / shape tier”。G2 real round-trip spec 只是检查 env 并 skip，`tests/e2e/specs/dogfood-providers.spec.ts:76-86`；G3 real LLM 同样 skip，`dogfood-multi-agent.spec.ts:95-110`；G5 明确 deferred。只有 G4 maintenance 相对接近真实。

W10：Design components 确实有 8 个，`apps/web/src/components/design/index.ts:11-39`。但 Design audit 不干净：`apps/web/src/app/(app)/docs/new/page.tsx:93-158` 仍有 `text-zinc-*`、`rounded-md`、`border-zinc-*`、`bg-zinc-900`；STATUS 说 docs/new 已迁移为 0 命中，这不成立。

我无法在当前只读沙箱跑通 `pnpm typecheck` 或包级测试：pnpm 在仓库根写 `_tmp_*` 时 EPERM，并触发 install 状态检查失败。这是环境权限失败，不代表测试本身失败。

覆盖质量判断：two-author e2e 覆盖 agent propose/reject/accept/export 和 permission boundary，`tests/e2e/specs/two-author-mvp.spec.ts:63-280`，但它是 HTTP-driven，明确不测 sync-gateway/y-sweet，`:19-21`。关键路径单测覆盖看起来不错：provider adapters 用 stub fetch，`packages/ai-runtime/tests/provider-byo.test.ts:78-175`；但 W9 dogfood spec 对真实 provider/multi-agent/subdoc stress 的覆盖偏“声明式”。

额外发现一个会直接影响 typecheck 的明显 bug：`apps/web/src/lib/inline-agent-menu.ts` 重复导出 `ChipMode`，`:29` 和 `:62`。STATUS 里提到的 Duplicate ChipMode baseline 仍在 HEAD。

## 6. Design SoT 合规扫描
`components/design/` 方向是对的，Button/MonoDisc/StatusPill/ProvenanceCard 等都 token-driven。但 `apps/web/src` 仍有 reject criteria 命中：`docs/new/page.tsx:100-158` 多处 `rounded-md`、`border-zinc`、`bg-zinc-900`；`Editor.tsx:117-128` 也还有 red/zinc filled 状态块，`:187` 还有 `prose-zinc ... rounded-md border-zinc-200 bg-white`。这说明 W10.7 audit 只清了部分 app surface，没有覆盖 package-exported editor-core React 组件和 docs/new。

trade-off：把所有 UI 都抽到 design components 会拖慢交付；但至少 “Design.md reject grep 0” 不能写进 STATUS，除非 grep 范围和例外清单固定。

## 7. 文档 / onboarding
README 比旧版本更清楚，但存在 overclaim：`README.md:7` 写“服务器关停后仍能编辑、渲染、导出”，当前离线编辑有 IndexedDB，离线渲染/导出没有完整客户端路径。SELF_HOST 明确预算“约 1 小时”，`docs/SELF_HOST.md:3-7`，不满足你要求的 30 分钟。

onboarding 5 步剧本存在，`apps/web/public/demo/onboarding.md:15-105`，但它自己承认步 3-5 真跑依赖 Phase 5 / external key，`:121-129`。ADR-INDEX 的 9×4 导航还算有用，但文件最后更新停在 2026-05-10，且 0012-0015 仍显示 Proposed，`plan0/ADR-INDEX.md:10`、`:29-32`，与 STATUS 的 Accepted-with-caveat 不一致。

## Top 5 Risks
1. **ADR-0008 runtime 不可中断/无 quota**：这是 P0，不应等 Phase 5 才补。
2. **STATUS 诚实度赤字**：多处 “PASS / Accepted” 实际是 mock、shape-only 或 deferred。
3. **ModelProvider 没接入主 invoke route**：设置 UI + resolver 有，但实际 agent invoke 仍 env-default Anthropic/mock。
4. **Subdocument 还没经过真实 sync-gateway 多 subdoc + ACL stress**：ADR-0014 只能算 backend scaffold。
5. **Design gate 声称清零但代码仍命中**：会削弱 Design.md 作为 SoT 的权威。

## Top 5 Wins
1. Provenance writer + approvalChain 是真实护城河。
2. Claim/Evidence + maintenance scan 方向正确，且多为 SQL-pure。
3. InlineAgentMenu 把 “AI as action” 从口号推进到了可用 surface。
4. CJK typography pre-pass 做得细，标点 boundary 已补。
5. Plugin provider contract 已从 Anthropic SDK shape 切开，第三方扩展边界比之前健康。

## 给项目所有者的 3 个具体建议
1. **先补 ADR-0008，不要再加新功能**：`quota-enforcer`、`cancel` route、worker polling、timeout enforcement 必须成为下一批 P0。
2. **把 dogfood gate 改成三态**：`contract PASS / mock PASS / real PASS`，STATUS 和 ADR promote 只允许 real PASS 支撑 Accepted。
3. **做一次硬 grep gate 修正**：固定 Design.md grep 范围为 `apps/web/src + packages/editor-core/src/Editor.tsx`，把 docs/new 和 Editor.tsx 的 reject 命中清掉，或在 STATUS 明确列为例外。
