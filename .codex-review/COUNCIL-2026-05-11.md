# Consciousness Council — 综合修改迭代方案

## Phase 1 — 议会召集

议题：codex 给出 5 risks + 3 建议（ADR-0008 quota / STATUS 诚实度 / ModelProvider 主路由 / Subdocument stress / Design 命中），项目当前在 Phase 4 closeout 后、ADR moratorium 期、Phase 5 未启动。**该不该按 codex 优先级专项补？怎么补？**

议会选 5 角色（生产性张力优先）：
1. **Architect** — 看结构 / 长期债
2. **Pragmatist** — 看本周能落地什么
3. **Contrarian** — 怀疑 codex 框架本身
4. **Strategist** — 看 Phase 5 二阶效应
5. **Minimalist** — 问哪些可以不修

## Phase 2 — 议事

### 🎭 The Architect

**Position**：codex 的 5 risks 里 **3 个是结构债，不补 Phase 5 必卡**，专项不可避免。

**Reasoning**：doc-store 暴露 `yDoc` escape hatch（`packages/doc-store/src/types.ts:50-55`）= ADR-0001 §5.D "1-2 周可切 Loro" 是空话；sync-gateway 没 multi-subdoc routing = ADR-0014 是 backend scaffold 不是同步系统；ADR-0008 没 quota enforcer + 没 cancel route = 没 backpressure 的系统不是系统。这三件**单独**都能扛，**组合**起来 Phase 5 fork-merge / plugin marketplace / multi-agent 任何一条都会撞墙。

**Key Risk They See**：所有人都低估"schema/同步协议钉死的代价"——CLAUDE.md §5.1 自己写过的红线，正在被自己破。

**Surprising Insight**：codex 没说出口但更危险的——`agent-worker reviewer/researcher 仍 markDone stub`（codex 引用 `apps/agent-worker/src/index.ts:132-149`）意味着 **ADR-0011 maintenance scan 在 reviewer/researcher 路径上是 mock**，W9 G4 标 "real PASS" 实际只覆盖部分 finding kind。这条 codex 也没抓全。

---

### 🎭 The Pragmatist

**Position**：1 周内把 5 risks 里的 4 条尾巴清掉，**ADR-0008 quota 缓做**（没用户、没流量、没钱烧）。

**Reasoning**：能算工时的只有 4 件：(a) ChipMode 重复导出删除 = 10 min；(b) `/api/agent/invoke` 接 ModelProvider resolver = 1 天；(c) STATUS PASS 拆三态 = 2 小时；(d) docs/new + Editor.tsx Design 命中清零 = 半天。Architect 说的 doc-store yDoc removal 是 2-3 周——这是 Phase 4.5 不是本周修。

**Key Risk They See**：Architect 让大家停一切做基础设施，Phase 5 W1 启动延后一个月，复利损失大于结构债利息。

**Surprising Insight**：codex 报告**自己有内部矛盾**——TL;DR 写"底层方向对"，但 honesty deficit table 给 5 个 ADR 打 P1。如果 5 个 ADR 都 P1，**底层不是对的，是欠债的**。codex 这点不诚实——它该写"底层方向对但兑现度 60%"。

---

### 🎭 The Contrarian

**Position**：**codex review 本身被 prompt bias 污染**——把 "honest caveat" 误读为 "deficit"——议会先做 cross-check 再谈优先级。

**Reasoning**：项目所有者在 STATUS 显式写 "Accepted with caveat" 是**诚实标注**，不是赤字。codex prompt 写"挑刺优先于赞美"，所以它把所有 caveat 都升级成 deficit——这是 reflexive effect，不是发现。Design 命中也类似：`docs/new/page.tsx` 是占位创建表单、`Editor.tsx:117-128` 是 editor-core 的 React 包——它们**本来就可能不在 Design SoT 范围**（Design SoT 是 `apps/web/src/components/design/`），用 grep 范围圈定即可。

**Key Risk They See**：codex 是"外部权威 + opus 模型"——项目所有者会权重过高，但它没在你的代码库里跑过测试，没看 git blame，对项目史一无所知。

**Surprising Insight**：codex 报告里**唯一硬确凿的发现是 ChipMode 重复导出 bug**——其他全是 framing 判断。把 codex 的 5 risks 当 5 个 hypothesis（不是 5 个 fact），用 typecheck / dogfood log / git history cross-check，至少 2 个会变软。

---

### 🎭 The Strategist

**Position**：开 **"Phase 4.5"，1.5 周专项**，只补 **跟 Phase 5 强耦合的 3 项**（doc-store yDoc removal / plugin sandbox macOS / ModelProvider 接主路由）。其余 deferred。

**Reasoning**：Phase 5 W1 候选 wave（multi-doc UI / fork-merge / plugin marketplace）的依赖图：
- multi-doc UI ← doc-store 不 leak yDoc
- plugin marketplace ← sandbox macOS 真做 + ADR-0008 quota（防第三方烧用户钱包）
- 任何 agent feature ← ModelProvider resolver 接主路由

ADR-0014 subdocument 反而**不是 blocker**——`improvement-plan-2026-05.md §四`已经把 spatial canvas / 章节 fork-merge UI / Loro 切换砍 / 推到 Phase 6+。所以 ADR-0014 deferred 是正确选择——codex 把它列 P1 是没读改进计划。

**Key Risk They See**：Pragmatist 想跳过 Phase 4.5 直接 Phase 5 W1，结果 W3 卡死回头补，单段成本 = 现在专项的 2-3 倍。

**Surprising Insight**：ADR-0008 quota 真正的 trigger 不是"long-horizon agent"，是"plugin marketplace 第三方烧钱"。所以 quota 应该跟 plugin sandbox macOS 绑在一起做（同一周内），不是独立 P0。

---

### 🎭 The Minimalist

**Position**：codex 5 risks 里 **2 件 + 1 删除** 即可结案——剩下都是 over-fitting 焦虑。

**Reasoning**：问"不修也能跑吗"：
| Risk | 不修后果 | 修复成本 |
|---|---|---|
| ADR-0008 quota | 没用户调用，零事故风险 | 1-2 周 |
| STATUS 诚实度 | 改文字 | 半小时 |
| ModelProvider 主路由 | Anthropic env 已工作 | 1 天 |
| Subdocument stress | Phase 5 才用 | 1 周 |
| Design 命中 | 肉眼不可见 | 半天 |
| ChipMode 重复导出 | 影响 typecheck | 10 min |

**做 3 件**：(a) ChipMode 删；(b) STATUS 加 "contract / mock / real" 三态；(c) `/api/agent/invoke` 一行 fallback 到 resolver。**不开 Phase 4.5**。

**Key Risk They See**：Architect / Strategist 把 "Phase 5 blocker" 当成必然，但 **Phase 5 启动条件本身没定义**——没第一个用户、没场景。在用户出现前补 blocker 是猜需求。

**Surprising Insight**：codex 报告自己列了 5 wins（Provenance / Claim/Evidence / InlineAgentMenu / CJK / Plugin contract）——**如果 80% 已对，修法应该是渐进的，不是专项的**。专项 = 暗示 80% 不对。

## Phase 3 — 综合

### ⚖️ Council Synthesis

**Points of Convergence**（3+ 角色 align）：
1. **ChipMode 重复导出立删**（Architect/Pragmatist/Minimalist）——10 min，无 trade-off
2. **STATUS PASS 拆三态 contract / mock / real**（Pragmatist/Strategist/Minimalist）——半小时，移除 codex 5 risks 里 4 条的"不诚实"指控
3. **`/api/agent/invoke` 接入 ModelProvider resolver**（Pragmatist/Strategist/Minimalist）——1 天，低成本高 leverage，把 ADR-0013 从"contract pass + 主路由 stub"升到"主路由真接"

**Core Tension**：
> **Architect+Strategist 阵营**："基础设施欠债是 Phase 5 blocker，必须 1.5 周专项 (Phase 4.5) 补 doc-store / sandbox macOS / quota"
> 对
> **Pragmatist+Minimalist 阵营**："没用户没事故，专项 = 过度工程；渐进修 + 把 codex risks 一半证伪即可"

差距 = 1.5 周工时 + Phase 5 启动延后/不延后。

**The Blind Spot**：
没人问 — **"Phase 5 启动 trigger 是什么？"** 第一个真实用户在哪？第一个场景是"两位作者写 bilingual paper" 还是 "实验室 maintain dataset citation"？**Phase 5 trigger 不定义，所有"Phase 5 blocker"讨论都是悬空**——包括 Architect / Strategist 的 P1 优先级。codex 没问，议会四个人也没问。

**Recommended Path**（吸纳张力，不消除）：

```
Day 0 (今天，4 小时)              ┐
  ├─ ChipMode 重复导出删除         │  Convergence
  ├─ STATUS PASS 拆三态            │  全员同意
  ├─ /api/agent/invoke 接 resolver │  低成本
  └─ 写 PHASE-5-TRIGGER.md         ┘  blind spot 补

Day 1-3 (诚实展示)
  └─ 选 1 件 P1 做"实证展示" → 推荐 doc-store yDoc removal
     - 理由：直接堵 ADR-0001 §5.D 的最大空话
     - 量化：完成后 STATUS 可以删掉 "1-2 周可切 Loro" caveat
     - 验收：grep `yDoc` in `packages/doc-store/src/` = 0
     - 预估：3 天（Strategist 估的 2-3 周打折——只 removal、不真接 Loro）

Day 4-5 (trigger 而非 blocker)
  └─ 找到第一个真实用户场景 + 写 PHASE-5-WAVE-A-SCOPE.md
     - 在 trigger 落地前不动 ADR-0008 quota / sandbox macOS / subdocument stress
     - 理由：这三项的"P1"全部依赖"将有用户调用"——用户没出现就是猜需求

Day 6+ (Phase 5 启动)
  └─ 按 trigger 反推 P1 真子集
     - 如果 trigger = bilingual paper → P1 = ModelProvider 多 provider 真 round-trip
     - 如果 trigger = plugin marketplace → P1 = sandbox macOS + quota
     - 如果 trigger = multi-doc 实验室 → P1 = sync-gateway multi-subdoc
```

**反对 Architect 全开 Phase 4.5 的理由**：3 项一起做 = 1.5 周，单做 doc-store removal = 3 天，**节省 1 周给 trigger 探索**——更高 ROI。
**反对 Minimalist 完全不动结构的理由**：doc-store yDoc removal 是 ADR-0001 直接承诺的硬话，不堵 = 项目长期诚实度受损。一件 ≠ 全做。
**反对 Pragmatist "ADR-0008 quota 缓做"的理由**：等 trigger，trigger 决定它什么时候起；不是缓做，是按需做。
**采纳 Contrarian 的 cross-check**：所有 codex risks 进 STATUS 之前，先 grep / typecheck / dogfood log 验证一遍——附在 STATUS §X "codex review 验证记录"。

**Confidence Level**：**Medium-High**。Convergence 段 3 件是高置信；Day 1-3 doc-store removal 选择是 medium（Architect 会说不够、Minimalist 会说多余）；blind spot (Phase 5 trigger) 是最高置信——四角色没问就是它。

**One Question to Sit With**：
> **你现在想优化的是"项目继续往前跑的速度"还是"项目对外可见的诚实度"？**
> 这两个偶尔冲突——选速度，doc-store yDoc 留着；选诚实度，专项补。**当前你已经把 STATUS / ADR caveat 写得相当诚实，所以"诚实度"边际收益在递减；速度的边际收益在递增**。但这是你的判断，不是议会的。
